Continuous Delivery

Continuously deliver software in small increments with automated testing and deployment to improve quality and speed of delivery.

Key aspects

  • Version Control
  • Work in Small Batches
  • Limit Work In-Progress (WIP)
  • Adhering to Coding Standards
  • Trunk-Based Development
  • Loosely Coupled Architecture
  • Continuous Integration
  • Feature toggles to decouple deployment from release
  • Shift Left on Quality
  • Test Automation
  • Test Data Management
  • Deployment Automation
  • Shift Left on Security and Privacy
  • Lightweight Change Approval
  • Proactive Notification on Build and Deployment Process
  • Monitoring and Obtaining Feedback from Production
  • Make Flow of Work Visible to Team and Customer
  • Visual Management of Current Application Production, Build, and Deployment State
  • Empowered Teams
  • Team Experimentation
  • Communication and Collaboration Skills
  • Continuous Learning and Skill Development
  • Agile Methodologies and Practices

PHR Guidelines / Work Instructions

CDG-1 — Automate Build and Deployment Processes

  1. Use build automation tools like Jenkins or GitLab CI/CD.
  2. Define build and deployment scripts and artifact versioning.
  3. Use IaC tools for environment provisioning.

CDG-2 — Implement Continuous Integration (CI)

  1. Run automated builds and tests on commits.
  2. Include unit, integration and acceptance tests in CI.
  3. Report test results and maintain code coverage goals.

CDG-3 — Establish Comprehensive Automated Testing

  1. Define a testing strategy covering unit, integration, and regression tests.
  2. Automate test execution and integrate with CI.
  3. Monitor flakiness and maintain test health.

CDG-4 — Adopt Infrastructure as Code (IaC)

  1. Version control infrastructure code and deploy reproducible environments.
  2. Automate provisioning and configuration.
  3. Test IaC changes before applying to production.

CDG-5 — Enable Continuous Deployment

  1. Use canary or blue-green deployments to reduce risk.
  2. Automate rollback strategies and monitor for anomalies.
  3. Ensure access controls around deployment actions.

CDG-6 — Implement Monitoring and Alerting

  1. Collect metrics, logs and traces from production.
  2. Define alerting thresholds and incident playbooks.
  3. Visualize key metrics in dashboards.

CDG-7 — Promote Collaboration and Communication

  1. Encourage cross-functional teams and knowledge sharing.
  2. Use communication tools and hold regular ceremonies.
  3. Practice blameless postmortems to learn from incidents.

CDG-8 — Establish Version Control Best Practices

  1. Use Git and trunk-based development where possible.
  2. Enforce code reviews and branch naming conventions.
  3. Use feature flags for dark launches.

CDG-9 — Implement Agile and DevOps Principles

  1. Adopt iterative development and continuous improvement.
  2. Automate routine tasks and foster shared responsibility.
  3. Track metrics to measure process improvements.

CDG-10 — Perform Regular Retrospectives and Process Refinements

  1. Run retrospectives after iterations to capture learnings.
  2. Prioritize process improvements and track progress.
