Shift Left on Security & Privacy

Integrate security and privacy early into the SDLC to reduce vulnerabilities and ensure privacy compliance.

Key aspects

  • Threat Modeling: Identify threats, assess impact, and plan mitigations early.
  • Secure Design: Apply secure-by-design principles and least privilege.
  • Code Analysis & Testing: Use static analysis, SAST/DAST and automated security tests.
  • Secure Code Review: Incorporate security-focused peer reviews and checklists.
  • Training: Provide security and privacy training for developers and stakeholders.
  • DevSecOps: Automate security checks within CI/CD pipelines.
  • Privacy by Design: Apply data minimization, consent handling, and secure data practices.

Risks of not implementing

RiskDescription
SLSPR-1Increased vulnerability exposure — more security gaps.
SLSPR-3Non-compliance with privacy regulations — legal and financial risks.
SLSPR-6Data leakage or loss — exposure of sensitive information.
SLSPR-10Increased business risks — reputational and financial impact.

Guidelines / Work Instructions

  1. SLSPG-1 - Threat Modeling: Run and update threat models during design and feature planning.
  2. SLSPG-2 - Secure Coding: Enforce secure coding rules and SAST checks.
  3. SLSPG-3 - Security Testing: Integrate SAST/DAST and pen-testing into CI/CD.
  4. SLSPG-4 - Secure Frameworks: Adopt secure development frameworks like OWASP SAMM.
  5. SLSPG-5 - Monitoring & Incident Response: Implement logging, monitoring, and an incident response plan.
  6. SLSPG-6 - Training: Keep teams trained and informed on threats and privacy requirements.
  7. SLSPG-7 - Privacy by Design: Build privacy controls and consent mechanisms into the product.
