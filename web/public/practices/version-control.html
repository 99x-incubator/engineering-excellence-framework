Version Control

A critical practice to manage and track source code changes

Overview

Version control is a critical best practice in software engineering that helps manage and track changes to source code and other project assets over time. It enables collaboration among developers, facilitates bug tracking, and provides a safety net for code changes.

Key aspects covered by the practice

  • Repository management: Establish a centralized repository to store the project's source code and related files. Use Git or similar systems.
  • Branching strategy: Use branches for independent development (e.g., GitFlow, GitHub Flow).
  • Commit frequently and meaningfully: Make granular commits with helpful messages.
  • Pull requests/code reviews: Use PRs for reviews before merging to main branches.
  • Commit history maintenance: Keep history clean for rollbacks/cherry-picks.
  • Merge and conflict resolution: Resolve conflicts promptly and correctly.
  • Version tagging: Tag releases (SemVer) to mark milestones.
  • Backup and disaster recovery: Backup repositories and verify recovery processes.
  • Documentation and collaboration: Use commit messages, issues and review comments to provide context.

Guidelines / Work Instructions

  1. VCG-1 - Establish a Centralized Repository: Choose Git, set up a repo, define structure and access controls.
  2. VCG-2 - Adopt a Branching Strategy: Define main branch, use feature branches, document chosen strategy.
  3. VCG-4 - Implement Code Review Process: Use PRs and define review guidelines.
  4. VCG-5 - Resolve Merge Conflicts Promptly: Train developers and document conflict resolution processes.
  5. VCG-6 - Utilize Version Tagging: Use SemVer and tag releases.
  6. VCG-7 - Regularly Backup the Repository: Automate backups and test recovery periodically.
  7. VCG-8 - Encourage Documentation and Collaboration: Use commit messages, integrate issue tracking and promote knowledge sharing.
