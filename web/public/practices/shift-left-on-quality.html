Shift Left on Quality

Move QA and quality practices earlier in the lifecycle to detect defects sooner and reduce rework.

Key aspects

  • Early Defect Detection: Involve QA from requirements to prevent defect propagation.
  • Collaboration: Encourage cross-functional collaboration between developers and QA.
  • TDD: Write tests before code to enforce quality-driven development.
  • Automation & CI: Automate tests and integrate them into CI pipelines for quick feedback.
  • Code Reviews & Static Analysis: Enforce regular code reviews and static checks.
  • Requirements Validation: Ensure requirements are clear, complete and testable.

Risks of not implementing

RiskDescription
SLQR-1Late Discovery of Critical Defects — costly fixes late in the lifecycle.
SLQR-2Increased Rework Effort — more time spent fixing defects.
SLQR-3Inadequate Test Coverage — missed defects due to late testing.
SLQR-4Lack of Alignment on Quality Goals — poor coordination between teams.

Guidelines / Work Instructions

  1. SLQG-1 - Early QA Involvement: Include QA in requirement workshops and planning.
  2. SLQG-2 - TDD: Adopt test-first practices and refactor based on test results.
  3. SLQG-3 - CI & Automated Testing: Integrate tests in CI pipelines and run them frequently.
  4. SLQG-4 - Code Reviews & Static Analysis: Enforce code quality checks and reviews.
  5. SLQG-5 - Requirements Validation: Define acceptance criteria and test cases up-front.
  6. SLQG-6 - Continuous Learning: Run retrospectives and evolve practices based on metrics and feedback.
